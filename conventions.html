P02 - Consolidated Pattern Library

File Naming Conventions

Folder Name

  • All the folder name should be in lowercase. (ex: img, css)

  • DO NOT type space in the folder name (ex: img folder)

CSS File Name

  • Name all css files in lowercase

  • DO NOT type space in the CSS file name (ex: main grid.css)

HTML File Name

  • Write capital letter in the begining of each words (ex: Home.html, Shop.html)

  • DO NOT type space in the HTML file name (ex: About Us.html)

  • Try to put all letter together instead of either spacing or "-" (ex: AboutUs.html)

  • Name HTML files same as their pages name. (ex: Home.html, Shop.html)

Class & ID Conventions Name

  • use "_" or "-" between each Character(ex:footer_socialMedia)

  • DO NOT name based on generics (ex: .button-1, .button-2)

  • Try to name class in specific way. (ex: .brown-btn, yellow-btn)

Image File Name

  • Name image files categorically (ex: logo.png, itemlist_1.jpg, health_icon.png)

File Structure Conventions

Folders

  • css: contains all css files, such as grid.css, pattern.css, or main.css

  • img: contains all image files, such as logos, icons, and product images

CSS Files

  • grid.css: contains all grid and layout Structure

  • main.css: contains reusable patterns, such as font styles, text colors, or button designs,

  • pattern.css: contains stylized patterns for modular elements, such as product lists, review stars, navigarion, and manymore

HTML Files

  • Structure HTML files with pages categories (ex: Home.html, AboutUs.html)
